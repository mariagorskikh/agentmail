AgentMail

An agent-native mailbox. Email, redesigned for a world where most of your inbound comes from software.

What is this?

A mailbox where every inbound message — from humans or from agents — is read, classified, and routed before it touches your attention.

Triage

Your inbox, triaged by an agent

Every inbound message is classified, lane-routed, drafted-when-appropriate, and audit-logged. You see what needs you, not everything.

Five lanes

Five lanes, not infinite unread

Quarantine, filed, auto-action, draft-for-review, escalated. Each message lands in exactly one. No more bottomless scroll.

Agent-native

Agents are first-class senders

Structured HMAC-authed POST, not just SMTP. Per-agent identity, per-agent rate limits, per-agent trust levels.

How it works

One pipeline. Five lanes. Every step audited. A 60-second undo window before anything leaves the building.

inbound message ▸ ingest ▸ classify ▸ route to lane 1..5 │ ▼ (lane 4) draft for human ▸ approve ▸ send (60s undo) │ ▼ (every step) audit log

Built for both sides

A mailbox is a contract between a sender and an owner. AgentMail makes that contract explicit on both ends.

For humans

You own a mailbox.

  • Only review what needs you — Lane 4 (drafts) and Lane 5 (escalated).
  • Everything else is filed, auto-actioned, or quarantined for you.
  • Full audit log of every classification, draft, and send.
  • 60-second undo window before any outbound message leaves.
Self-host → Open inbox →

For agents

Send structured messages to humans without burning email reputation.

  • HMAC authentication — no SMTP, no SPF gymnastics.
  • Per-agent rate limits, scopes, and trust levels.
  • Predictable lane assignment based on topic + trust.
  • Poll /api/v1/agent/messages/{id} for status — received, drafted, sent, rejected.
  • The human is always in the loop for anything that matters.

Register an agent

Mailbox owners issue agent keys. Each key has its own identity, trust level, and rate budget. The secret is shown once, at creation.

Owner authentication

Paste your API token. We store it locally in your browser so you don't have to re-enter it.

Sent as Authorization: Bearer <token>. Kept in localStorage as am_admin_token.

New agent

Each agent gets a stable agent_id, a display name, and a freshly generated secret.

Lowercase, digits, hyphens. 3–41 chars. Must start alphanumeric.
Higher trust = more aggressive lane routing. Start with unknown.

My agents

List active agents on this mailbox. Secrets are not shown — only the key prefix.
